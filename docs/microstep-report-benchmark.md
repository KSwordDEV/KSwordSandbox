动态分析 + 失败/补充信息”的结构。KSwordSandbox 的 HTML 报告应逐步
对齐以下章节：

1. 封面与样本身份信息。
2. 目录。
3. 行为检测。
4. 多维检测。

8. 执行流程。

## 样本身份字段

参考报告封面和静态页覆盖了多种文件身份字段。KSwordSandbox v1 已有
`SampleIdentity` 的文件名、路径、SHA-256 和大小；后续应扩展：

- MD5、SHA-1、CRC32；
- SSDEEP、TLSH、peHashNG、impfuzzy；
- 文件格式和 Magic 类型；
- PE 位数、子系统、入口点、节区数量；
- 标签，例如 `packer`、`tls_callback`、异常节名、加密算法线索；
- 首次提交、末次提交、末次分析等情报字段，如果未来接入本地或远程
  情报库。

## 检测摘要字段

参考报告把检测结论压缩成几个高层指标。KSwordSandbox 的报告首页应
生成这些聚合结果：

- 总体判定：正常、可疑、高危、失败。
- 风险分组：高危行为、可疑行为、通用行为。
- MITRE ATT&CK 命中技术数。
- Yara 或静态规则命中数。
- 引擎检出率或本地规则命中率。
- 威胁分类和家族字段，初期可留空或显示“未分类”。

## 行为检测示例方向

提取结果中的行为摘要集中在文件落地、加壳和 PE 异常。对应到本项目
可先落地这些规则方向：

- 在用户目录创建可执行文件。
- 在临时目录创建文件。
- 在文件系统上释放或创建新的可执行文件。
- 可执行文件疑似加壳或压缩。
- UPX 压缩特征。
- PE 节大小或节名异常。

这些规则可以来自三类证据：

- Guest Agent 的文件差异事件，例如 `file.created`、`file.modified`。
- R0 文件 minifilter 事件，例如释放文件、重命名、覆盖写。
- 静态分析器的 PE 节区、入口点、导入表、字符串和 Yara 命中。

## 多维检测与 MITRE 映射

参考报告展示 MITRE ATT&CK 技术命中数量，并将行为按风险等级组织。
KSwordSandbox 规则模型应保持以下字段：

- `RuleId`：规则稳定编号。
- `Title`：面向报告的行为标题。
- `Severity`：`high`、`medium`、`low`、`info`。
- `MitreTechniqueId` 和 `MitreTechniqueName`。
- `Summary`：简短解释。
- `Evidence`：命中的原始事件列表。

当前 `rules/behavior-rules.json` 已包含脚本执行、注册表修改、文件落地
和网络连接的种子规则；后续需要增加 PE 静态规则、进程树规则、持久化
规则和反分析规则。

## 静态分析目标

参考报告静态页包含基础信息、哈希、PE 元数据、标签、字符串和 URL。
KSwordSandbox 后续应新增 host-side 静态分析管线：

1. 文件哈希：SHA-256、SHA-1、MD5、CRC32。
2. PE 解析：架构、子系统、入口点、节区、导入表、导出表、TLS 回调。
3. 字符串提取：ASCII、UTF-16、URL、路径、注册表键、命令行片段。
4. 加壳判断：UPX、异常节名、入口点落在异常节区、节熵。
5. Yara 扫描：初始样本、释放文件、内存转储。
6. 特征解释：将静态命中映射到报告可读描述。

## 动态分析目标

参考报告动态页至少预留了执行流程、进程详情、截图和网络行为。结合
本项目 v1 方案，动态页应覆盖：

- 进程树：父子进程、命令行、启动/退出时间、退出码。
- 线程和镜像加载：来自 R0 回调或驱动事件。
- 文件行为：创建、修改、删除、重命名、释放可执行文件。
- 注册表行为：创建、设置、删除、启动项、服务项。
- 网络行为：DNS、HTTP、HTTPS、TCP、UDP、域名、IP、端口、SNI/URL。
- 截图：运行期间按时间点截屏；v1 可先预留字段和展示槽位。
- 失败原因：VM 不存在、凭据缺失、驱动未加载、样本启动失败、超时、
  事件为空、回收日志失败。

## 网络行为字段

参考报告将网络行为拆分为指纹、域名、DNS、HTTP、HTTPS、TCP、UDP。
KSwordSandbox v1 的 Guest Agent 当前只做 TCP 差异快照；R0/WFP 接入后
应扩展为：

- DNS 查询和响应；
- TCP/UDP 连接五元组；
- HTTP 方法、Host、URI、User-Agent、状态码；
- TLS SNI、证书摘要；
- 域名/IP 去重统计；
- 可疑网络 IOC 表。

## 报告美观和可读性要求

参考报告面向人工阅读，强调摘要优先、证据可展开。KSwordSandbox HTML
报告应采用：

- 顶部风险摘要卡片；
- 左侧或顶部目录导航；
- 行为按风险等级分组；
- 每条行为可展开证据事件；
- 静态与动态分栏；
- 表格支持长路径、哈希和命令行换行；
- 失败信息单独突出展示。
